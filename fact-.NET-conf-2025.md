กรณี Session .NET โดย Boat Charunthon
ข้อสังเกตเชิงเทคนิคเกี่ยวกับความสอดคล้องของเนื้อหาในการนำเสนอ Session .NET ดังกล่าว โดยมีวัตถุประสงค์เพื่อความโปร่งใส ความถูกต้องทางวิชาการ

1. ประเด็นด้านความสอดคล้องของข้อมูล (Consistency Issue)
จากการตรวจสอบสไลด์การนำเสนอ พบว่ามีการระบุการวัด benchmark บนโมเดลขนาด 1.5B parameters
อย่างไรก็ตาม ในการสาธิตและการรันจริงโดยผู้บรรยาย กลับเลือกใช้โมเดลขนาด 0.5B parameters
ความไม่สอดคล้องระหว่าง ข้อมูลที่ระบุในสไลด์ กับ การทดสอบที่เกิดขึ้นจริง อาจทำให้ผู้ฟังเข้าใจคลาดเคลื่อนเกี่ยวกับผลลัพธ์และข้อสรุปของการ benchmark

2. ความเหมาะสมของโมเดลที่ใช้ในการ Benchmark
โดยทั่วไป โมเดลขนาด 0.5B parameters มีข้อจำกัดด้านความสามารถอย่างมีนัยสำคัญ และยังไม่เป็นที่ยอมรับอย่างแพร่หลายทั้งในเชิงธุรกิจ (Business Use Case) และเชิงเทคนิค (Production / Applied AI)
ดังนั้น การนำผลลัพธ์จากโมเดลดังกล่าวมาใช้แทนหรืออ้างอิงแทนโมเดลขนาด 1.5B อาจไม่สะท้อนประสิทธิภาพหรือข้อจำกัดของระบบได้อย่างเหมาะสม

3. ผลกระทบต่อความน่าเชื่อถือของการนำเสนอ
การนำเสนอผล benchmark ที่อิงจากโมเดลซึ่งมีขนาดและคุณสมบัติแตกต่างจากที่ระบุไว้ อาจเข้าข่ายการสื่อสารที่ไม่ครบถ้วน และส่งผลต่อความน่าเชื่อถือของเนื้อหาในเชิงวิชาการและวิชาชีพ
ประเด็นดังกล่าวจึงควรได้รับการชี้แจงอย่างชัดเจน เพื่อให้ผู้ฟังและผู้อ่านสามารถประเมินความถูกต้องของข้อมูลได้อย่างเป็นธรรม
